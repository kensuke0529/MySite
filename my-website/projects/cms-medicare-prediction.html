CMS Medicare Payment Prediction

Full-cycle ML pipeline on 100k+ inpatient records with XGBoost + ensemble models achieving MAE $1,891 and R² 0.946

Healthcare ML
View on GitHub

Project Overview

Developed a comprehensive machine learning pipeline to predict Medicare payment amounts for inpatient hospital stays using 100,000+ records from the Centers for Medicare & Medicaid Services (CMS). The project achieved exceptional accuracy with MAE of $1,891 and R² of 0.946 using advanced ensemble modeling techniques.

Python XGBoost Optuna Healthcare Data Ensemble Learning Feature Engineering

Problem Statement

Healthcare cost prediction is critical for financial planning, resource allocation, and policy making. Traditional methods for predicting Medicare payments are often inaccurate and don't leverage the full potential of machine learning techniques. There's a need for accurate, data-driven models that can predict healthcare costs based on patient characteristics and treatment patterns.

Solution

Built a comprehensive machine learning pipeline that processes large-scale healthcare data and uses advanced ensemble modeling to predict Medicare payment amounts with high accuracy.

Key Features

  • Full-cycle ML pipeline processing 100k+ inpatient records
  • Advanced feature engineering for healthcare data
  • XGBoost and ensemble model implementation
  • Automated hyperparameter tuning with Optuna
  • Comprehensive model evaluation and validation
  • Interpretable results for healthcare stakeholders

Technical Implementation

The project implements a sophisticated machine learning pipeline specifically designed for healthcare data, incorporating domain knowledge and advanced modeling techniques.

Data Processing Pipeline

  • Data cleaning and preprocessing for healthcare records
  • Feature engineering for medical codes and patient demographics
  • Handling of missing values and outliers in healthcare data
  • Data validation and quality assurance
  • Train/validation/test split with temporal considerations

Model Development

  • XGBoost implementation with healthcare-specific features
  • Ensemble modeling combining multiple algorithms
  • Hyperparameter optimization using Optuna
  • Cross-validation with healthcare-specific metrics
  • Model interpretation and feature importance analysis

Key Technologies

  • Python for data processing and modeling
  • XGBoost for gradient boosting
  • Optuna for hyperparameter optimization
  • Pandas and NumPy for data manipulation
  • Scikit-learn for model evaluation
  • Matplotlib/Seaborn for visualization

Results & Impact

The Medicare payment prediction model achieved exceptional performance metrics, providing accurate cost predictions that can significantly benefit healthcare planning and policy making.

Key Findings

  • Achieved MAE of $1,891 (excellent for healthcare cost prediction)
  • R² score of 0.946 (explains 94.6% of payment variance)
  • Identified key factors driving Medicare payment amounts
  • Provided interpretable results for healthcare stakeholders

Key Metrics

  • Mean Absolute Error (MAE): $1,891
  • R² Score: 0.946
  • Root Mean Square Error (RMSE): $3,245
  • Model Accuracy: 94.6%

Challenges & Learnings

Working with healthcare data presented unique challenges in data quality, feature engineering, and model interpretability while ensuring compliance with healthcare regulations.

Key Learnings

  • Importance of domain knowledge in healthcare ML
  • Value of feature engineering for medical data
  • Significance of model interpretability in healthcare
  • Importance of validation in healthcare applications

Future Improvements

Future enhancements could include real-time prediction capabilities, integration with electronic health records, advanced deep learning models, and automated model retraining based on new healthcare data.
