Service Ticket Classification System

AI & NLP BERT PyTorch
Project Overview

This project implements a multi-level ticket classification system using BERT-based models to categorize service tickets into main categories and hardware subcategories. The system was designed to automate IT support ticket routing and improve response times, achieving 97.78% accuracy on main classification and 95.56% on hardware subcategories.

Dataset & Model

The system was trained on a dataset of 1,200 tickets with text lengths ranging from 36 to 181 characters. The data was split into 70% training, 15% validation, and 15% testing sets.

  • Main Categories: Hardware, Network, Database, Software, Inquiry/Help
  • Hardware Subcategories: CPU, Disk, Keyboard, Memory, Monitor, Mouse
  • Base Model: BERT-base-uncased
  • Framework: PyTorch with Transformers library
  • Tokenization: Max length 128 tokens
Key Findings

The model demonstrated excellent overall performance with both classification tasks achieving >95% accuracy. Early stopping effectively prevented overfitting, leading to strong generalization on the test set.

  • Strengths: High precision and recall across most categories.
  • Areas for Improvement: The 'Software' category had the lowest F1-score (0.93), often confused with Hardware/Network issues.
  • Business Impact: 97.78% accuracy enables reliable automated ticket routing, significantly reducing manual classification time.
Technical Implementation

The project was implemented using PyTorch and the Hugging Face Transformers library. The BERT-base-uncased model was fine-tuned for both main category and hardware subcategory classification tasks.

  • Technologies: PyTorch, Hugging Face Transformers, BERT, Python.
  • Training: AdamW optimizer (lr=2e-5), Early stopping (patience=5).
